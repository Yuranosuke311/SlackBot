## 4. 型定義

### 4.1 `types/intern-salary.ts`

```typescript
// インターン生のプロフィール（不変情報・更新可）
export type InternProfile = {
  slack_user_id: string;
  name: string;
  phone: string;
  bank_info: string;    // "〇〇銀行 〇〇支店 普通 1234567"
  unit_price: number;   // 円/時
  updated_at: string;   // ISO 8601
};

// 月次提出データ
export type InternSalarySubmission = {
  id: string;                   // UUID v4
  intern_id: string;            // Slack User ID
  intern_name: string;
  month: string;                // "YYYY-MM"
  unit_price: number;           // 円/時
  working_hours: number;        // 稼働時間（小数可）
  expense_transport: number;    // 移動費（円）
  expense_travel: number;       // 交通費（円）
  expense_ai: number;           // AI利用費（円）
  total_salary: number;         // unit_price × working_hours
  total_expense: number;        // expense_transport + expense_travel + expense_ai
  total_amount: number;         // total_salary + total_expense
  bank_info: string;
  payment_due: string;          // "YYYY-MM-DD"（来月末日）
  status: "submitted" | "paid";
  submitted_at: string;         // ISO 8601
  paid_at: string | null;
};

// 確認待ち一時データ（確定/修正で削除）
export type InternSalaryPending = Omit<InternSalarySubmission, "id" | "status" | "submitted_at" | "paid_at"> & {
  pending_since: string; // ISO 8601
};
```

## 5. データ設計（Upstash Redis）

### 5.1 Redis キー設計

| キー | 型 | 内容 |
|---|---|---|
| `intern:profile:{slack_user_id}` | String (JSON) | `InternProfile` |
| `intern:known_users` | Set | 過去に1回以上提出した Slack User ID |
| `intern:salary:{id}` | String (JSON) | `InternSalarySubmission` |
| `intern:salary:months:{YYYY-MM}` | Set | その月の提出 ID 集合 |
| `intern:salary:submitted_users:{YYYY-MM}` | Set | その月に提出済みの Slack User ID 集合 |
| `intern:salary:pending:{slack_user_id}` | String (JSON) | `InternSalaryPending`（確認待ち一時データ） |

### 5.2 計算ルール

```
total_salary  = unit_price × working_hours（端数切り捨て）
total_expense = expense_transport + expense_travel + expense_ai
total_amount  = total_salary + total_expense
payment_due   = 来月の末日（例: 4月提出 → 5月31日）
```

## 7. API エンドポイント設計

### 7.1 `POST /api/cron/intern-notify`（新規）

**役割**: cron-job.org から毎朝9時（JST）に呼び出される日次処理

**認証**: `Authorization: Bearer {CRON_SECRET}`

**処理フロー**:

```
1. Authorization ヘッダー検証
2. today（JST "YYYY-MM-DD"）を取得
3. 今月の "YYYY-MM" と来月末日を計算

■ ケース A: 月末3日前〜月末日（インターン生リマインド）
  - getKnownInternIds() で全既知インターン生 ID を取得
  - getSubmittedUserIds(currentMonth) で提出済み ID を取得
  - 未提出 ID = 既知 ID - 提出済み ID
  - 未提出の各インターン生に DM リマインドを送信（ボタン付き）

■ ケース B: 来月末3日前〜来月末日（佐藤さんへサマリー）
  - getSubmissionsForMonth(currentMonth) で提出一覧を取得
  - getKnownInternIds() - getSubmittedUserIds() = 未提出者リスト
  - MANAGER_SLACK_ID に DM でサマリー送信（提出一覧 + ✅ボタン + 未提出者リスト）

4. HTTP 200 を返す
```

**日付計算ユーティリティ**:

```typescript
function getTodayJST(): string  // "YYYY-MM-DD"
function getCurrentMonthJST(): string  // "YYYY-MM"
function getLastDayOfMonth(year: number, month: number): string  // "YYYY-MM-DD"
function getLastDayOfNextMonth(): string  // "YYYY-MM-DD"
function getDaysUntil(targetDate: string, today: string): number
```

#### ケース E: モーダル送信・確認DM送信（`callback_id: "intern_salary_modal"`）

```
1. 署名検証

3. 数値バリデーション（単価・稼働時間・各経費が正の数値か）
4. 金額計算
   total_salary  = floor(unit_price × working_hours)
   total_expense = expense_transport + expense_travel + expense_ai

7. chat.postMessage で確認 DM を送信（ブロック内に confirm_intern_salary / edit_intern_salary ボタン）
```

## 8. Cron ジョブ詳細設計

### 8.1 `app/api/cron/intern-notify/route.ts`

```typescript
export async function POST(request: NextRequest) {
  // Bearer 認証
  const authHeader = request.headers.get("authorization");
  if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }

  const today = getTodayJST();                    // "YYYY-MM-DD"
  const currentMonth = getCurrentMonthJST();      // "YYYY-MM"
  const lastDayOfMonth = getLastDayOfMonth(...);  // "YYYY-MM-DD"
  const lastDayOfNextMonth = getLastDayOfNextMonth(); // "YYYY-MM-DD"

  const daysUntilSubmission = getDaysUntil(lastDayOfMonth, today);
  const daysUntilPayment = getDaysUntil(lastDayOfNextMonth, today);

  // ■ ケース A: 提出期限3日前〜当日 → 未提出インターン生へリマインド
  if (daysUntilSubmission >= 0 && daysUntilSubmission <= 3) {
    const knownIds = await getKnownInternIds();
    const submittedIds = await getSubmittedUserIds(currentMonth);
    const unsubmittedIds = knownIds.filter(id => !submittedIds.includes(id));

    for (const userId of unsubmittedIds) {
      // DM リマインド送信
    }
  }

  // ■ ケース B: 支払期限3日前〜当日 → 佐藤さんへサマリー通知
  if (daysUntilPayment >= 0 && daysUntilPayment <= 3) {
    const submissions = await getSubmissionsForMonth(currentMonth);
    const knownIds = await getKnownInternIds();
    const submittedIds = await getSubmittedUserIds(currentMonth);
    const unsubmittedIds = knownIds.filter(id => !submittedIds.includes(id));

    // 佐藤さんへサマリー DM 送信
  }

  return NextResponse.json({ ok: true });
}
```

## 10. エラーハンドリング方針

| 状況 | 対応 |
|---|---|
| 署名検証失敗 | 403 を返す |
| CRON_SECRET 不一致 | 401 を返す |
| 数値入力が不正（文字など） | モーダルバリデーションエラーを返す（`response_action: "errors"`） |
| Pending データが存在しない | スキップ・200 を返す（タイムアウト等の冪等性確保） |
| getSubmission で null | スキップ・200 を返す |
| Slack API エラー | ログ出力・Cron は次の処理を継続 |

### モーダルバリデーションエラーの返し方

```typescript
// interactions/route.ts 内
if (isNaN(unitPrice) || unitPrice <= 0) {
  return NextResponse.json({
    response_action: "errors",
    errors: {
      unit_price_block: "正の数値を入力してください"
    }
  });
}
```

## 11. Slack アプリ追加設定

### 11.1 追加 OAuth スコープ

| スコープ | 用途 | 追加要否 |

| `chat:write` | メッセージ送信・更新 | フェーズ1で設定済み |

2. `/invite @BotName` で Bot を招待
3. チャンネル ID を `INTERN_SALARY_CHANNEL_ID` に設定

## 12. cron-job.org 追加設定

既存の `notify` Cron に加えて以下を追加:

|---|---|
| URL | `https://{your-app}.vercel.app/api/cron/intern-notify` |

| 実行スケジュール | 毎日 00:00 UTC（= 09:00 JST） |
| ヘッダー | `Authorization: Bearer {CRON_SECRET}` |

### 13.2 動作確認チェックリスト

- [ ] チャンネルのボタンクリック → モーダルが開く
- [ ] 初回入力 → 空フォーム、2回目 → 前回値が自動入力される
- [ ] 「内容を確認する」→ 確認 DM が届く（金額計算が正しいか確認）
- [ ] 「✅ 確定して送信」→ `MANAGER_SLACK_ID`（= 自分）に通知が届く
- [ ] 「✏️ 修正する」→ 入力値を引き継いだモーダルが再表示される
- [ ] 「✅ 振込完了」→ メッセージが完了表示に書き換わる

```bash
# Cron 手動実行

  -H "Authorization: Bearer {CRON_SECRET}"
```
